# Documentation Principles

## Principles

- route before you repeat
- keep labels stable and repository-owned
- let the root site explain the network, not replace repository handbooks
- preserve the shared shell unless a repository has a clear structural reason to extend it

## Review Lens

- can a reader choose the right repository within a few seconds
- do hub labels still match the labels used by the destination sites
- does each page make the next click obvious without duplicating deep handbook detail
